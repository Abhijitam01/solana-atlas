# RFC Process

This repository uses lightweight RFCs for proposals that impact multiple templates, learning flows, or execution behavior.

## When to write an RFC
- New template families or curriculum tiers
- Changes to template schemas
- Execution engine or runner behavior changes
- New analytics or cohort features

## How to submit
1. Copy the template: `docs/rfcs/000-template-submission.md`
2. Create `docs/rfcs/NNN-short-title.md`
3. Open a PR and request review from the core maintainers

## Acceptance criteria
- Clear problem statement
- Proposed design and scope
- Impact on templates and learning flow
- Rollout plan and risks
